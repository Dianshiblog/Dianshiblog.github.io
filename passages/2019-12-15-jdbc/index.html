Java （L2）- JDBC

JDBC作为Java与关系型数据库交互的接口，在开发中扮演着十分重要的角色。

Part I. 简介

JDBC 即 Java Database Connectivity， 中文名为 Java 数据库连接， 是由 Sun 公司定义的一组 API 。 各大数据库厂商根据定义做了具体的实现，并提供jar 驱动包给用户使用，当使用 JDBC 进行编程时，执行的是 jar 包中的实现类。

数据库、JDBC、驱动、Java 代码的关系如下图所示：

java.sql 包提供了 JDBC 的类与接口，主要有以下几种：

  • Driver 接口
  • Connection 接口
  • Statement 接口
  • PreparedStatement 接口
  • CallableStatement 接口
  • ResultSet 接口

Part II. 快速入门

接下来以 MySQL 为例，介绍 JDBC 快速入门：

  1. 导入驱动 jar 包，添加到项目的 libs 目录下；
  2. 注册驱动，MySQL 5 及之后版本的驱动可省略此步骤；
  3. 获取数据库连接对象 Connection；
  4. 获取执行 sql 语句的对象 Statement；
  5. 定义 sql 语句；
  6. 执行 sql 语句，返回结果；
  7. 处理结果并释放资源。

Part VI. 数据库连接池

上文中每次连接数据库都需要加载数据库驱动、获取 Connection 对象、获取 Statement 对象、执行 sql 语句、释放资源等步骤。而建立数据库连接是一件比较消耗资源和时间的事情，若使用者较多会给系统造成比较大的负担。一个常见的改良方法是创建一个用来存放数据库连接的容器，容器初始化时会申请一些连接对象，用户对数据库进行访问时会先容器中获取一个连接对象，用户访问完成后会将类，连接对象归还给容器。

0.数据库连接池的实现

标准接口：javax.sql 包中的 DataSource 接口。

  • getConnection() 获取连接对象
  • Connection.close() 如果 Connection 对象为数据库连接池获取的，则调用 close() 方法可以归还连接对象。

数据库连接池的实现：由数据库厂商实现，目前用用比较广的是以下两个：

  • C3p0
