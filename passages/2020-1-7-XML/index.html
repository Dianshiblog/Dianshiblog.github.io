今天学习的内容是 XML 的用法，包括快速入门、语法和解析方法

Part I. 介绍与快速入门

  1. 概念

    XML 全称为 Extensible Markup Language，即可扩展的标记语言，特点是可自行定义标签，常用于配置文件中。

    XML 与 HTML 同出自 w3c (万维网联盟)，而 XML 的语法是十分严格的，HTML 的语法比较松散。

  2. 快速入门

    特点：

    • 属性值必须使用单引号或者双引号括起来
    • 标签必须关闭

Part III. XML 约束

XML 约束规定了 XML 文档的标签书写规则，需要学会如何在 XML 中引入约束。

目前用的比较多的有 DTD 和 Schema 两种。

  1. DTD 约束
    • 内部 DTD：将约束规则定义在 XML 文档中
    • 外部 DTD：将约束规则定义在外部 dtd 文件中
      • 本地 dtd 文件：第二行书写
      • 网络 dtd 文件：第二行书写
  2. Schema 约束
    • <① ② ③ ④>
      • ① 根标签名
      • ② 引入 xsi 前缀，如 xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
      • ③ 引入 xsd 文件命名空间 xsi:schemaLocation="http://www.dianshi.cn/xml student.xsd"
      • ④ 为 xsd 约束声明前缀作为标识 xmlns="http://www.dianshi.cn/xml"

Part IV. XML 解析

解析 XML 文件即将其加载到内存中。

常用的解析 XML 的方式：

  • DOM：将 XML 文件一次性加载到内存中，形成一棵 DOM 树进行解析。操作方便，可以进行 CRUD 操作，但比较占内存。
  • SAX：基于事件驱动，逐行读取 XML 文件。内存占用低，但是只能读取，不能增删改。

常见的 XML 解析器：

  • JAXP：Sun 官方提供的 Java XML 解析器，不太好用。
  • DOM4J
  • Jsoup：提供了一组 Java API，并且可以直接解析 URL 地址和 HTML 文件
  • PULL：Android 内置的 XML 解析器
  1. 下面着重介绍 Jsoup 的使用：
  • 获取 XML 文件的路径
  • 利用 Jsoup.parse 对路径的 XML 文件进行解析，获得 Document 对象
  • 利用 Document 对象通过 getElementsByxxx() 方法获得 Element 或 Elements（元素是 Element 的 ArrayList），并获取数据

element.attr(String key) ：根据属性名称获取属性值

element.text() ：获取文本内容

element.html() ：获取标签体的所有内容(包括字标签的字符串内容)

  1. 快速查询

    • selector:选择器

      使用的方法：Elements select(String cssQuery)

    • XPath

      XPath即为XML路径语言，它是一种用来确定XML（标准通用标记语言的子集）文档中某部分位置的语言，使用之前需要导入依赖 jar 包。
